    GroupBy

    Bom depois dever muita função lambda você deve se perguntar se não existe um jeito melhor de obter aqueles resultados, existe uma função muito boa chamada groupby ela basicamente evita os filtros , maps e reduces que fizemos, porém não ache que o conhecimento de map, filter e reduce podem ser descartados, são ótimas funções para trabalhar com coleções.

    Primeiramente vamos entender oque é e oque faz a função groupby, como o próprio nome diz ela serva para agrupar dados de acordo com alguma característica, sintaticamente a função aceita 1 ou 2 parâmetros, o primeiro deles é o iterável, ou seja, nossa coleção de dados e o segundo parâmetro opcional é a característica pela qual queremos agrupar, como dito podemos escolher não passar essa característica e neste caso a groupby vaio agrupar pelos próprios elementos da coleção, vamos ver logo um código com groupby.

    from itertools import groupby

frase = 'AAAAAAABBBBBCCCDDDEEA'

listaAgrupada = [list(valor) for item, valor in groupby(frase)]
print(*listaAgrupada)

array=[1,1,1,2,3,4,67,23,12,1,2,2,1]
listaAgrupada = [list(valor) for item, valor in groupby(array)]
print(*listaAgrupada)

    NOTE QUE: Tivemos que realizar a importação de groupby da biblioteca itertools

    Galera se executarmos o código acima vamos perceber que o agrupamento ocorreu de maneira estranha, por exemplo quando estávamos agrupando a frase e escrevemos no console as lista com os itens repetidos aparecem duas listas com o valor 'A', isso acontece pois sempre que vamos utilizar o groupby a coleção deve estar ordenada, isso porque a implementação da função groupby nos pede isso, veremos um trecho da implementação dela abaixo:

    while self.currkey == self.tgtkey:
    self.currvalue = next(self.it)    # Exit on StopIteration
    self.currkey = self.keyfunc(self.currvalue)
self.tgtkey = self.currkey

    Implementação completa: itertools — Functions creating iterators for efficient looping — Python 3.9.5 documentation

    Vamos tentar explicar para você o funcionamento do trecho de código acima, não se assuste com self veremos essa palavra reservada quando estivermos falando sobre classes, basicamente temos um loop que verifica se curkey, ou seja, chave corrente é igual a tgtkey a "chave alvo", lembram que nós dissemos que se não passarmos a chave / característica para agrupar a função considera o próprio elemento como a característica, pois bem quando "AAAAAAABBBBBCCCDDDEEA" entrou na função a tgtkey = 'A' entramos naquele loop while e fomos até a primeira ocorrência de uma curkey diferente de 'A', ou seja, o valor 'B', note que não chegamos até o ultimo 'A' da frase e por isso ele está em um grupo diferente, concluindo acreditamos que você entendeu o porque devemos ordenar a coleção antes de aplicarmos groupby, vamos corrigir os exemplos abaixo:

    from itertools import groupby

frase = 'AAAAAAABBBBBCCCDDDEEA'

listaAgrupada = [list(valor) for item, valor in groupby(sort(frase))]
print(*listaAgrupada)

array=[1,1,1,2,3,4,67,23,12,1,2,2,1]
listaAgrupada = [list(valor) for item, valor in groupby(sort(array))]
print(*listaAgrupada)

    Pronto agora temos o resultado esperado para o agrupamento, dito isso e feita as devidas correções vamos para aquele exercício que deixou você maluco com map, reduce e filter, agora vocês vão ver como o groupby é poderoso quando bem utilizado, voltamos a dizer que os conhecimentos adquiridos sobre map, reduce e filter são muito importantes independente do seu conhecimento relativo a groupby.
